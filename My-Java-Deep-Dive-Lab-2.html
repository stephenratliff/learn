1. Modern JS Setup ES2024

Write code for today's engines, not 2012. Modules, strict mode, and devtools are your base camp.

Modern browsers ship ES2022–2024 features by default. That means class fields, top-level await, Array.fromAsync, Object.groupBy, and logical assignment are safe. Use <script type="module"> — modules are deferred, strict by default, and have their own scope. No more accidental globals.

Senior tip: Treat every file as a module. It enforces good boundaries, enables tree-shaking, and prevents hoisting surprises.
hello.mjs
// Modules are strict, deferred, and scoped
export const greet = (name = 'World') => `Hello ${name}!`;

// Top-level await works in modules (ES2022)
const delay = ms => new Promise(r => setTimeout(r, ms));
await delay(10);

console.log(greet('Stephen'));

// ES2024: Object.groupBy
const langs = ['js','ts','rs','js','ts'];
console.log(Object.groupBy(langs, l => l));
Under the hood: Module scripts are fetched in parallel, parsed once, and executed in dependency order after HTML parsing. No var leaks to window. Use importmap for bare specifiers in the browser without bundling.

DevTools console power moves

  • $_ – last result. $0 – currently selected element.
  • copy(obj), console.table(arr), monitor(fn)
  • Use console.time('loop') / timeEnd for quick profiling.

2. Core Language Deep Dive

Hoisting, scope, closures, this, and coercion — the four traps that still bite seniors.

let / const / var

var is function-scoped and hoisted with undefined. let/const are block-scoped and live in the TDZ until initialized. Prefer const by default; use let only for reassignment.

hoisting.js
console.log(a); // undefined (hoisted)
var a = 1;

// console.log(b); // ReferenceError TDZ
let b = 2;

Scope chain & closures

A closure is a function that remembers its lexical environment. Used for privacy, factories, and memoization — zero global state.

closureCounter.js
function createCounter(start=0){
  let count = start; // private
  return {
    inc: () => ++count,
    dec: () => --count,
    get: () => count
  };
}
const c = createCounter(5);
console.log(c.inc(), c.inc(), c.get()); // 6 7 7

The four this rules

  1. Default: undefined in strict mode.
  2. Implicit: obj.method() → this = obj
  3. Explicit: call/apply/bind
  4. new: this = new instance. Arrow functions ignore all and use lexical this.
thisDemo.js
const user = {
  id: 42,
  getId(){ return this.id },
  getIdArrow: () => this?.id
};
console.log('implicit:', user.getId()); // 42
console.log('arrow:', user.getIdArrow()); // undefined (lexical)

const unbound = user.getId;
try { console.log(unbound()) } catch(e){ console.log('lost this') }

const bound = user.getId.bind({id:99});
console.log('bind:', bound()); // 99
Common pitfall: passing methods as callbacks loses this. Fix with bind, or define as arrow field in class.

3. Functions and Advanced Patterns

Functions are values. Master composition over inheritance, and your code shrinks.

memoize.js
const memoize = (fn) => {
  const cache = new Map();
  return (...args) => {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const val = fn(...args);
    cache.set(key, val);
    return val;
  };
};

const fib = memoize(n => n<2 ? n : fib(n-1)+fib(n-2));
console.log(fib(40)); // instant after first run
curry.js
const curry = fn => (...a) => a.length >= fn.length ? fn(...a) : (...b) => curry(fn)(...a,...b);
const add = (a,b,c) => a+b+c;
const add3 = curry(add);
console.log(add3(1)(2)(3)); // 6

// Rest/spread + destructuring
const logUser = ({id, name='anon', ...rest}) => console.log(id, name, rest);
logUser({id:1, role:'admin'});
Why it matters: Memoization saves CPU for pure functions. Currying enables partial application for pipelines. Rest/spread avoids manual loops and mutation.

4. Objects, Prototypes, and Classes

Prototype chain is still the engine. Classes are sugar — use them with private fields for true encapsulation.

prototypeChain.js
const person = { greet(){ return `hi ${this.name}` } };
const stephen = Object.create(person);
stephen.name = 'Stephen';
console.log(stephen.greet()); // hi Stephen
console.log(Object.getPrototypeOf(stephen) === person); // true
BankAccount.js
class BankAccount {
  #balance = 0; // true private
  constructor(owner){ this.owner = owner }
  deposit(amt){ this.#balance += amt; return this }
  withdraw(amt){ if(amt>this.#balance) throw new Error('NSF'); this.#balance-=amt; return this }
  get balance(){ return this.#balance }
}
const acct = new BankAccount('Stephen');
acct.deposit(100).withdraw(30);
console.log(acct.balance); // 70
Composition > inheritance: Favor small objects with methods that delegate. Use Object.assign or spread for mixins. Inheritance chains hurt maintainability.

fetchWithRetry.js
async function fetchWithRetry(url, {retries=3, timeout=3000} = {}){
  for(let i=0;i<=retries;i++){
    const ac = new AbortController();
    const t = setTimeout(()=>ac.abort(), timeout);
    try {
      const res = await fetch(url, {signal: ac.signal});
      clearTimeout(t);
      if(!res.ok) throw new Error(res.status);
      return await res.json();
    } catch(e){
      if(i===retries) throw e;
      await new Promise(r=>setTimeout(r, 2**i*200)); // exponential backoff
    }
  }
}
fetchWithRetry('https://jsonplaceholder.typicode.com/todos/1')
  .then(d=>console.log('ok', d.title))
  .catch(e=>console.log('failed', e.message));

        
Use Promise.allSettled for independent requests. Cancel with AbortController. Never mix callbacks and promises.

9. Functional and Reactive Patterns

        
Immutability + pure functions = predictable UI.

        
dataPipeline.js
const users = [
  {id:1, active:true, score:42},
  {id:2, active:false, score:17},
  {id:3, active:true, score:88},
];

const pipe = (...fns) => x => fns.reduce((v,f)=>f(v), x);

const result = pipe(
  arr => arr.filter(u => u.active),
  arr => arr.map(u => ({...u, grade: u.score>80?'A':'B'})), // immutable
  arr => arr.sort((a,b)=>b.score-a.score),
  arr => arr.flatMap(u => [u.id, u.grade])
)(users);

console.log(result); // [3,'A', 1,'B']

      


      

        
10. Error Handling and Debugging

        
safeParse.js
class ValidationError extends Error { constructor(msg, cause){ super(msg); this.name='ValidationError'; this.cause=cause } }

function safeParse(json){
  try { return JSON.parse(json) }
  catch(e){ throw new ValidationError('Bad JSON', {cause:e}) }
}

try { safeParse('{bad}') } catch(e){
  console.error(e.name, e.message);
  console.log(e.cause?.cause?.message);
}

        
Swallowing errors is tech debt. Always log with context, and use finally for cleanup.
