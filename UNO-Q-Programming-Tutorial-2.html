Getting Started with the Arduino UNO Q

A beginner's guide to programming hardware with C++ and Python.

Part 1: The C++ Sketch (Controlling the MCU)

An Arduino "sketch" is the name for a program written for the microcontroller. It's written in C++. Every sketch has two main functions:

  • void setup() { ... }: This code runs once when the board first powers on. You use it for initialization, like telling the board which pins you're going to use.
  • void loop() { ... }: This code runs over and over again, forever. This is where you put the main logic of your program, like blinking an LED or reading a sensor.

Your First Sketch: Blinking an LED

This is the "Hello, World!" of hardware. It will blink the built-in LED on the board. This code is written in the C++ sketch file (e.g., sketch.ino).

Goal: Make the onboard LED (connected to pin 13) turn on for one second, then off for one second, repeatedly. 
// This code runs on the MCU (the real-time brain)

const int LED_PIN = 13; // The built-in LED is on pin 13

void setup() {
  // This function runs once at the start.
  pinMode(LED_PIN, OUTPUT); // Set the LED pin as an output.
}

void loop() {
  // This function runs in a loop, forever.
  digitalWrite(LED_PIN, HIGH); // Turn the LED on. HIGH is voltage ON.
  delay(1000);              // Wait for 1000 milliseconds (1 second).
  digitalWrite(LED_PIN, LOW);  // Turn the LED off. LOW is voltage OFF.
  delay(1000);              // Wait for another second.
}

Part 2: Python on the MPU (The Linux Brain)

The MPU side runs Python. From here, you can't directly control the hardware pins. Instead, you use the Arduino Bridge library to talk to the C++ sketch running on the MCU.

Project: Control the LED from Python

Let's modify the sketch so it waits for a command from Python before blinking the LED.

Step 1: The C++ Sketch to Listen for Commands

// C++ Sketch: sketch.ino
#include <Bridge.h>

const int LED_PIN = 13;

void setup() {
  pinMode(LED_PIN, OUTPUT);
  Bridge.begin();
  Bridge.on("blink", blinkHandler); // Register a handler for the "blink" command
}

void loop() {
  Bridge.process(); // Check for incoming commands from Python
}

void blinkHandler() {
  digitalWrite(LED_PIN, HIGH);
  delay(500);
  digitalWrite(LED_PIN, LOW);
}

Step 2: The Python Script to Send Commands

# Python Script: main.py
from arduinoio import Bridge
import time

bridge = Bridge()
bridge.begin()

print("Python script started...")

while True:
  bridge.call("blink") # Call the "blink" handler in the C++ sketch
  print("Sent 'blink' command.")
  time.sleep(2)

Part 3: Reading Sensor Data (C++ to Python)

A huge part of hardware is reading input from the physical world. Here, we'll read a value from a simple sensor (like a dial/potentiometer) and send that data back to our Python script for processing.

Hardware Needed: A potentiometer. Connect the middle pin to A0 on the Arduino, and the other two outer pins to GND and 5V.

Step 1: The C++ Sketch to Read and Return Data

This sketch registers a handler called "read_sensor". When Python calls it, the MCU will read the analog pin A0 and send the value back.

// C++ Sketch: sketch.ino
#include <Bridge.h>

const int SENSOR_PIN = A0; // Potentiometer on analog pin A0

void setup() {
  Bridge.begin();
  analogReadResolution(12); // Use 12-bit resolution for readings from 0-4095
  Bridge.on("read_sensor", readSensorHandler);
}

void loop() {
  Bridge.process();
}

// This function runs when Python calls "read_sensor"
void readSensorHandler() {
  int sensorValue = analogRead(SENSOR_PIN);
  // Send the value back to the Python script that called it.
  Bridge.respond(String(sensorValue));
}

Step 2: The Python Script to Request and Print Data

This Python script calls the "read_sensor" handler every second and prints the value it gets back.

# Python Script: main.py
from arduinoio import Bridge
import time

bridge = Bridge()
bridge.begin()

print("Polling sensor value from MCU...")

while True:
  # Call the handler and wait for the response from the MCU
  response = bridge.call("read_sensor")
  
  try:
    sensor_value = int(response.strip())
    print(f"Sensor value from A0: {sensor_value}")
    # Now you can use this value for anything in Python!
    if sensor_value > 4000:
      print("  -> High value detected!")
  except (ValueError, AttributeError):
    print("Could not parse response from MCU.")
  
  time.sleep(1)

Part 4: Full Project - Interactive LED Brightness

Let's combine everything. We will read the value from the potentiometer and use it to control the brightness of an LED. Since we need to pass a value (the brightness level) from Python to C++, we'll use a slightly different Bridge command.

Step 1: The C++ Sketch for Brightness Control

This sketch can receive a value along with the command.

// C++ Sketch: sketch.ino
#include <Bridge.h>

const int LED_PIN = 9; // Use pin 9, a PWM pin for brightness

void setup() {
  pinMode(LED_PIN, OUTPUT);
  Bridge.begin();
  Bridge.on("set_brightness", setBrightnessHandler);
}

void loop() {
  Bridge.process();
}

// The Bridge passes the value from Python into this function
void setBrightnessHandler(const String& value) {
  int brightness = value.toInt();
  analogWrite(LED_PIN, brightness); // analogWrite sets brightness (0-255)
}

Step 2: The Complete Python Control Script

This Python script reads the sensor value from A0 (using the handler from Part 3!) and then uses that value to tell the sketch in Part 4 what brightness to set.

# Python Script: main.py
# You will need BOTH C++ sketches from Part 3 and 4 loaded on the MCU.
# The App Lab IDE handles this automatically.
from arduinoio import Bridge
import time

bridge = Bridge()
bridge.begin()

print("Starting interactive LED control loop...")

while True:
  # 1. Read the sensor value from the MCU
  response = bridge.call("read_sensor")
  sensor_value = int(response.strip())
  
  # 2. Map the sensor value (0-4095) to a brightness value (0-255)
  # This is a classic Arduino technique called 'mapping'
  brightness = int((sensor_value / 4095) * 255)
  
  print(f"Sensor: {sensor_value} -> Brightness: {brightness}")
  
  # 3. Send the new brightness value to the MCU
  bridge.call("set_brightness", str(brightness))
  
  time.sleep(0.1) # Run this loop quickly for a smooth effect

Part 5: Quick Reference - Which Brain Does What?

When starting a new project, always ask: which brain is best for this task?

Task Best Brain Language Why?
Blinking an LED at a precise interval MCU C++ The MCU is built for real-time, precise timing.
Reading a sensor value MCU C++ The hardware pins are physically connected to the MCU.
Connecting to Wi-Fi / fetching web data MPU Python The MPU runs Linux and handles all networking.
Making decisions based on time of day MPU Python Python has powerful date/time libraries.
Running a web server to show sensor data MPU Python Use a Python web framework like Flask or FastAPI.
Controlling a motor with high precision MCU C++ Direct, real-time control is critical for motors.
