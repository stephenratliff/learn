Boxing: Every primitive has a wrapper class — int↔Integer, double↔Double, boolean↔Boolean. Autoboxing converts automatically, but watch out for NullPointerException when unboxing a null wrapper.
Variables.java
public class Variables {
    public static void main(String[] args) {
        // ── PRIMITIVES ────────────────────────────
        int     age      = 30;
        long    population= 8_000_000_000L;     // underscores OK
        double  pi       = 3.14159;
        float   price    = 19.99f;
        char    grade    = 'A';
        boolean isActive = true;

        // ── VAR (type inference, Java 10+) ────────
        var name = "Alice";        // inferred as String
        var count = 10;            // inferred as int

        // ── FINAL (constant, like 'const') ────────
        final double TAX_RATE = 0.08;

        // ── REFERENCE TYPES ────────────────────────
        String message = "Hello";     // object, not primitive
        Integer boxed  = 42;          // autoboxed int
        int unboxed     = boxed;        // auto-unboxed

        // ── CASTING ────────────────────────────────
        double d = 9.99;
        int truncated = (int) d;   // 9 — explicit narrowing
        long big = truncated;       // implicit widening

        System.out.println("Age: " + age);
    }
}

Classes & Objects

A class is a blueprint; an object is an instance. Java is purely object-oriented — every method lives inside a class, with no free-floating functions.

INTERACTIVE — Object Instances
simulated output
Dog.java
public class Dog {
    // ── FIELDS (instance state) ──────────────
    private String name;
    private int age;
    private static int totalDogs = 0;  // class-level

    // ── CONSTRUCTOR ───────────────────────────
    public Dog(String name, int age) {
        this.name = name;
        this.age = age;
        totalDogs++;
    }

    // ── METHODS ────────────────────────────────
    public String bark() {
        return name + " says Woof!";
    }

    public void haveBirthday() { age++; }

    // ── OVERRIDE Object methods ───────────────
    @Override
    public String toString() {
        return String.format("Dog{name='%s', age=%d}", name, age);
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Dog d)) return false;
        return age == d.age && name.equals(d.name);
    }

    public static int getTotalDogs() { return totalDogs; }
}

// ── USAGE ──────────────────────────────────
Dog rex = new Dog("Rex", 3);
System.out.println(rex.bark());
System.out.println(rex);                // uses toString()
System.out.println(Dog.getTotalDogs()); // static call

§ 07

Constructors

Special methods that initialize a new object. Java supports overloading — multiple constructors with different parameter signatures.

Constructors.java
public class Point {
    private final double x, y;

    // ── PRIMARY CONSTRUCTOR ───────────────────
    public Point(double x, double y) {
        this.x = x;
        this.y = y;
    }

    // ── OVERLOADED CONSTRUCTOR ────────────────
    public Point() {
        this(0.0, 0.0);    // delegates via this()
    }

    public Point(Point other) {     // copy constructor
        this(other.x, other.y);
    }

    // ── INSTANCE INITIALIZER BLOCK ────────────
    {
        System.out.println("New point created"); // runs before ctor body
    }

    // ── STATIC INITIALIZER (runs once, class load) ─
    static { System.out.println("Point class loaded"); }
}

// ── USAGE ──────────────────────────────────
Point origin = new Point();          // 0,0
Point p      = new Point(3, 4);
Point copy   = new Point(p);

// ── BUILDER PATTERN (common for many params) ──
public class Pizza {
    private final String size;
    private final boolean cheese;
    private Pizza(Builder b) { size=b.size; cheese=b.cheese; }

    public static class Builder {
        private String size = "medium";
        private boolean cheese = false;
        public Builder size(String s)   { size=s; return this; }
        public Builder cheese(boolean c){ cheese=c; return this; }
        public Pizza build()              { return new Pizza(this); }
    }
}
Pizza p2 = new Pizza.Builder().size("large").cheese(true).build();

public class Animal {
    protected String name;
    public Animal(String name) { this.name = name; }
    public String speak()  { return name + " makes a sound"; }

public class Dog extends Animal {
    private String breed;

    public Dog(String name, String breed) {

§ 09

Stack vs Heap

Primitives and references live on the call stack. Objects themselves are allocated on the heap, managed by the garbage collector.

VISUAL — Memory Layout
↑ Stack (method frames)
int age = 30
Dog rex = → 0x7F2A
Dog copy = → 0x7F2A (same ref!)
int[] nums = → 0x9B11
☁ Heap (objects, GC managed)
0x7F2ADog{ name="Rex", age=3 }
0x9B11int[] { 1, 2, 3, 4 }
Key insight: Dog copy = rex; doesn't clone the object — it copies the reference. Both variables point to the same heap object. Mutating through one is visible through the other.
memory.java
Dog rex  = new Dog("Rex", 3);  // heap object created
Dog copy = rex;             // copies the REFERENCE, not the object
copy.haveBirthday();         // mutates the SAME object
System.out.println(rex.getAge()); // 4 — rex also changed!

// Pass-by-value ALWAYS (even for objects — the reference is copied)
void reassign(Dog d) { d = new Dog("New", 0); }  // no effect outside
void mutate(Dog d)   { d.haveBirthday(); }       // DOES affect caller's object

// Garbage collection — objects with no references get GC'd
Dog temp = new Dog("Temp", 1);
temp = null;  // no more references → eligible for GC

§ 10

Interfaces & Abstract Classes

Interfaces define a contract — pure behavior with no state. Abstract classes can mix concrete and abstract methods, plus hold state.

Flyable.java
public interface Flyable {
    void fly();                  // implicitly abstract

    default String altitude() {  // default method (Java 8+)
        return "Cruising altitude";
    }

    static Flyable createDefault() { // static method
        return () -> System.out.println("Flying!");
    }
}

public interface Swimmable {
    void swim();
}

// Multiple interface implementation
public class Duck implements Flyable, Swimmable {
    @Override public void fly()  { System.out.println("Duck flies"); }
    @Override public void swim() { System.out.println("Duck swims"); }
}
Shape.java
public abstract class Shape {
    protected String color;       // state allowed!

    public Shape(String color) { this.color = color; }

    public abstract double area();  // must override

    public String describe() {       // concrete method
        return color + " shape, area=" + area();
    }
}

public class Circle extends Shape {
    private double radius;
    public Circle(String c, double r) { super(c); radius=r; }

    @Override
    public double area() { return Math.PI * radius * radius; }
}

// Cannot do: new Shape("red") — abstract!
Shape s = new Circle("red", 5);
System.out.println(s.describe());
Interface vs Abstract class: a class can implement many interfaces but extend only one class. Use interfaces for "can-do" capabilities, abstract classes for "is-a" hierarchies sharing state.

§ 12

Generics

Generics give you compile-time type safety for classes and methods that work over any type, eliminating manual casting and ClassCastException at runtime.

Generics.java
// ── GENERIC CLASS ──────────────────────────
public class Box<T> {
    private T content;
    public void set(T content) { this.content = content; }
    public T get()         { return content; }
}
Box<String> stringBox = new Box<>();
stringBox.set("hello");
String val = stringBox.get();     // no cast needed!

// ── GENERIC METHOD ─────────────────────────
public static <T> T firstElement(List<T> list) {
    return list.get(0);
}

// ── BOUNDED TYPE PARAMETERS ────────────────
public static <T extends Comparable<T>> T max(List<T> list) {
    T max = list.get(0);
    for (T item : list)
        if (item.compareTo(max) > 0) max = item;
    return max;
}

// ── WILDCARDS ──────────────────────────────
public static void printAll(List<?> list) { // unknown type
    for (Object o : list) System.out.println(o);
}
public static double sum(List<? extends Number> list) { // upper bound
    double total = 0;
    for (Number n : list) total += n.doubleValue();
    return total;
}

// ── MULTIPLE TYPE PARAMETERS ───────────────
public class Pair<K, V> {
    private final K key;
    private final V value;
    public Pair(K key, V value) { this.key=key; this.value=value; }
}
Pair<String, Integer> p = new Pair<>("age", 30);

File I/O

The modern java.nio.file package (NIO.2) offers a cleaner API than legacy java.io for reading, writing, and walking the filesystem.

FileIO.java
import java.nio.file.*;
import java.util.List;

// ── READ ENTIRE FILE ───────────────────────
Path path = Path.of("data.txt");
String content = Files.readString(path);
List<String> lines  = Files.readAllLines(path);

// ── WRITE ──────────────────────────────────
Files.writeString(path, "Hello, file!");
Files.write(path, lines, StandardOpenOption.APPEND);

// ── STREAM A LARGE FILE LINE-BY-LINE ──────
try (var stream = Files.lines(path)) {
    stream.filter(l -> !l.isBlank())
          .map(String::trim)
