Your Hardware: MacPro6,1

The Late 2013 Mac Pro — the iconic cylindrical "trash can" — packs a pair of AMD FirePro GPUs around a shared thermal core in a form factor that still turns heads. Despite Apple officially ending support at Ventura 13, the underlying hardware is genuinely capable of running Sonoma 14 with the right driver injection. The machine was designed to last, and it will.

Caveats & Limitations — Read First

⚠
Unsupported Configuration
Apple does not support this setup. Genius Bar and AppleCare service may be declined for systems running unsupported macOS versions. You assume responsibility for maintaining your patched installation.
🔐
SIP Is Partially Disabled
Root patching requires lowering System Integrity Protection so OCLP can write into the sealed system volume. AMFI is also modified. Your system will not pass Apple's standard security posture checks. For most workstation users this is an acceptable tradeoff.
🔄
Root Patches Must Be Re-Applied After Every macOS Update
Each Sonoma point release (14.x → 14.x+1) re-seals the system volume, erasing your GPU driver patches. After every update: launch OCLP → Post-Install Root Patches → Start Root Patching → Reboot. Skipping this leaves you in software-rendering mode.
📡
Rapid Security Responses Are Blocked
Apple's in-between mini security patches (RSRs) are incompatible with OCLP-patched systems and will be automatically blocked. Full point-release updates work normally.
✦
Why Sonoma Is the Smart Choice
Sonoma 14 is the version where OCLP support for MacPro6,1 is most mature. The team has had over a year of user feedback and patch refinement specifically targeting this macOS version on this hardware. Dozens of OCLP releases have been tested against Sonoma. The AMD FirePro root patches are reliable and well-documented. You are not on the bleeding edge.

Back Up Everything

The installation process writes to your internal SSD's EFI partition and reinstalls the operating system. If something goes wrong between steps, your drive could be in a partially modified state. A current backup is the only way to recover.

Time Machine (Required Minimum)

  1. Connect an external drive at least as large as your internal SSD.
  2. Open System Settings → General → Time Machine. Select your external drive as the backup destination.
  3. Click Back Up Now and wait for the full backup to complete. Don't cancel partway through.
  4. Verify: click the Time Machine icon in the menu bar → Browse Time Machine Backups. Navigate to a few recent files and confirm they're accessible.

Bootable Clone (Strongly Recommended)

A bootable clone is a complete, immediately usable copy of your current system on a separate external drive. If anything in the install process fails, you can boot from the clone and be back to work in under a minute, with zero data loss.

Use Carbon Copy Cloner or SuperDuper! to create one. After cloning, test it: hold Option (⌥) at startup, select the clone drive, and confirm it boots fully before proceeding.

✓
Test Before You Trust
An untested backup is hope, not a backup. Actually boot from your clone drive — or restore a single test file from Time Machine — and confirm it works before touching the installation process.

Build and Install OpenCore

OCLP detects your Mac Pro 6,1 automatically and builds a tailored OpenCore EFI — a custom bootloader that sits between the Mac's firmware and macOS, injecting the patches, hardware spoofing, and boot arguments needed to run Sonoma. Install it to a USB drive first. You can move it to the internal drive later once you've confirmed everything works.

🔧 Hardware Spoofing
OpenCore presents the MacPro6,1 as a supported Mac model so the Sonoma installer and system services don't reject the hardware.
📦 Kext Injection
Legacy kexts (kernel extensions) for AMD GCN GPU support are staged for post-install root patching.
⚙ Boot Arguments
Arguments like amfi_get_out_of_my_way and ipc_control_port_options=0 are injected at boot to allow OCLP's modifications.

Building OpenCore and Writing to a USB Drive

  1. Open OpenCore-Patcher.app. The main window shows your detected model: MacPro6,1. Click "Build and Install OpenCore".
  2. OCLP builds a custom OpenCore EFI for your specific configuration (10–30 seconds). Watch the build log for any errors — there should be none on a standard MacPro6,1.
  3. When the build succeeds, click "Install to Disk". A drive picker appears.
  4. Select your USB flash drive. OCLP writes the EFI to the USB drive's hidden EFI partition without erasing its data partition.
  5. Confirm: you'll see "OpenCore has been successfully installed to [USB drive name]."
ℹ
OCLP Auto-Configures Everything for MacPro6,1
The builder handles: hardware model ID spoof, AMD Legacy GCN driver path, NVMe boot support, USB port mapping, Thunderbolt 2 passthrough, NVRAM settings, and all required boot arguments. No manual config.plist editing is needed for standard MacPro6,1 hardware.
💾
If You Have a Non-Standard GPU
If you have installed an eGPU or swapped the original AMD FirePro cards for a different GPU model, OCLP may not auto-configure correctly. Stick with the original D300/D500/D700 FirePro cards for this guide; eGPU configurations require additional customization.

Create the macOS Sonoma Installer

You need a bootable Sonoma installer on a USB drive (16 GB minimum; 32 GB recommended if you want OpenCore and the installer on the same drive). OCLP can create this for you, or you can do it manually via Terminal.

Method A — Via OCLP GUI (Easiest)

  1. In OCLP's main window, click "Create macOS Installer".
  2. Choose "Download macOS Installer". Select macOS Sonoma from the list — pick the latest 14.x point release.
  3. OCLP downloads the full installer from Apple's servers (approximately 13 GB). This takes 20–60 minutes.
  4. Once downloaded, click "Flash Installer to Drive". Select your 16 GB+ USB drive. This erases the drive.
  5. Enter your administrator password. Flashing takes 5–15 minutes.

Method B — Via Terminal (If You Already Have the Installer)

If you downloaded the Install macOS Sonoma app from the App Store previously, create the USB installer with:

Terminal
# Replace /Volumes/MyVolume with your USB drive's volume name
% sudo /Applications/Install\ macOS\ Sonoma.app/Contents/Resources/createinstallmedia \
    --volume /Volumes/MyVolume

# Expected output:
# Erasing disk: 0%... 10%... 20%... 100%
# Copying essential files...
# Making disk bootable...
# Install media now available at "/Volumes/Install macOS Sonoma"
⚠
Always Use the Latest Sonoma Installer
If your installer was downloaded months ago, it may be an older 14.x build that is now incompatible with the latest OCLP root patches. Re-download a fresh copy when in doubt — use OCLP's built-in downloader to guarantee a current version.

Install macOS Sonoma

🔌
Both USB Drives Must Be Connected
You need the OpenCore USB and the Sonoma Installer USB plugged in simultaneously. OpenCore boots first; from within its picker you then select the Sonoma installer. Remove neither drive until installation is complete.

Booting Into OpenCore

  1. Plug in both USB drives. Power on (or restart) your Mac Pro.
  2. Hold Option (⌥) immediately at startup. The macOS Startup Manager shows available boot volumes as hard drive icons.
  3. Select the entry labeled "EFI Boot" (your OpenCore USB). Press Return.
  4. The OpenCore boot picker appears — a black screen with icons for each bootable volume. Select "Install macOS Sonoma" (your installer USB). Press Return.
🖥
The Screen Will Look Wrong — That's Expected
Before root patches are applied, your AMD FirePro GPUs run in software-rendering mode. Text may be fuzzy, resolution may be wrong, and animations will be slow. The installer itself will still work correctly. Proceed normally.

Running the Sonoma Installer

  1. The macOS installer environment loads. Select your language when prompted.
  2. Erase your destination first (recommended for a clean install): Open Disk Utility from the Utilities menu. Select your internal SSD. Click Erase. Format: APFS, Scheme: GUID Partition Map. Name it (e.g. Macintosh HD). Click Erase, then quit Disk Utility.
  3. Select "Install macOS Sonoma". Choose your internal SSD as the destination. Click Install.
  4. The installer copies files (≈5 minutes), then reboots. Keep both USB drives plugged in. When the OpenCore picker reappears, select the macOS Installer stage 2 entry. OCLP may auto-select it; if it does, leave it.
  5. The Mac Pro will reboot 2–4 more times during installation. Each time, the OpenCore picker will appear — select the installer stage entry. It will eventually boot into the macOS Sonoma Setup Assistant.
  6. Complete the Setup Assistant: region, keyboard layout, Wi-Fi, Apple ID, privacy settings, etc.
After setup is complete: do not install any software, do not run Software Update. Apply root patches first — the system is currently running on software rendering.
Post-Installation Phase
[10]

Apply Root Patches — The Critical Step

Root patching re-injects the legacy AMD GCN GPU driver stack into Sonoma's sealed system volume, giving your FirePro D-series cards their full Metal acceleration back. Without this step your Mac Pro is borderline unusable as a desktop. With it, performance is excellent.

📥 What Gets Injected
Legacy AMDRadeonX4000 and related GCN-architecture kernel extensions (kexts) for the FirePro Hawaii GPU core.
⚡ What You Gain
Full Metal GPU acceleration, proper display resolution, hardware video decode, QuickLook, Spotlight previews, full UI compositing.
🔄 When Required Again
After every macOS Sonoma point update (14.x → 14.y). Each update re-seals the system volume, removing the patches.

Step-by-Step Root Patching

  1. Boot into your new macOS Sonoma via the OpenCore picker. Log in.
  2. Transfer OpenCore-Patcher.app to this new Sonoma installation. Options: AirDrop from another Mac, copy from a USB drive, or re-download from GitHub.
  3. Launch OpenCore-Patcher.app.
  4. Click "Post-Install Root Patches".
  5. OCLP analyzes your system. For MacPro6,1, the patch list will include:
    • AMD Legacy GCN (Graphics Core Next) kexts
    • AMD Legacy Polaris/Vega kexts (may appear; only relevant ones are applied)
    • Miscellaneous patching for AMFI and SIP configuration
  6. Click "Start Root Patching". Enter your administrator password.
  7. OCLP performs these operations in sequence:
    • Remounts system volume as writable (read-write)
    • Copies legacy kexts into /System/Library/Extensions/
    • Rebuilds the kernel extension cache (kmutil)
    • Creates a new system snapshot and re-seals the volume
  8. When complete, click Reboot. Via the OpenCore picker, boot into macOS Sonoma.
  9. After login, the display should be at native resolution with full GPU acceleration active.

Confirming GPU Acceleration is Active

Terminal — GPU Check
# Confirm GPU model and Metal support
% system_profiler SPDisplaysDataType | grep -E "Chipset|VRAM|Metal"

# Expected output for D700 (example):

Handling Future Sonoma Updates

The golden rule: update OCLP before you update macOS, always.

Every Sonoma point release (14.x) re-seals the system volume, removing your root patches. Following the correct sequence every time keeps your Mac Pro running with full GPU acceleration. Doing it out of order risks being stuck in software-rendering mode until you can re-patch.

The Safe Update Sequence — Every Time

  1. A software update notification appears. Close it. Do not update yet.
  2. Go to github.com/dortania/OpenCore-Legacy-Patcher/releases and look for a release whose notes mention the new macOS version. Download it if available.
  3. Open the new OCLP → Build and Install OpenCore → install to your boot drive (USB or internal EFI, wherever you have OpenCore). This updates the bootloader.
  4. Now apply the macOS update: System Settings → General → Software Update → Update Now.
  5. The Mac Pro reboots through the update (OpenCore handles the boot transitions). Once on the desktop of the new Sonoma version: launch OCLP → Post-Install Root Patches → Start Root Patching.
  6. Reboot one final time. GPU acceleration is restored.

Going USB-Free: Permanent OpenCore on the Internal Drive

Once you've confirmed Sonoma runs correctly via the USB OpenCore stick — GPU acceleration active, all tests passing — you can install OpenCore permanently onto the internal SSD's EFI partition. The Mac Pro will then boot Sonoma automatically, with no USB drive required.

  1. Confirm you are booted into Sonoma with root patches applied and all hardware verified.
  2. Open OpenCore-Patcher.app → Build and Install OpenCore.
  3. In the Install to Disk dialog, select your internal SSD (e.g., APPLE SSD AP0512) instead of the USB drive.
  4. Confirm the selection. OCLP writes the OpenCore EFI to the internal EFI partition.
  5. Restart. The Mac Pro should now show the OpenCore picker automatically, with a 5-second countdown before booting Sonoma.
  6. Optionally suppress the picker for instant boots: In OCLP → Settings → set ShowPicker to Off. You can still access it anytime by holding Escape or Space at power-on.
Terminal — Verify Internal EFI Contents
# List disks to find your internal SSD disk identifier
% diskutil list

# Mount its EFI partition (substitute disk0 if needed)
% sudo diskutil mount disk0s1

# Confirm OpenCore EFI is present
% ls /Volumes/EFI/EFI/
# Should show: APPLE   BOOT   OC

# Unmount when done
% sudo diskutil unmount /Volumes/EFI
💾
Keep the USB OpenCore Stick
Even after installing OpenCore internally, keep your USB OpenCore stick in a drawer. If something ever goes wrong with the internal EFI — a failed update, a corrupted snapshot — you can boot from the USB and recover without needing any other hardware.

Troubleshooting

Holding Option (⌥) at startup doesn't show OpenCore / EFI Boot

The OpenCore USB may not have been written correctly. Confirm by plugging the USB into a different Mac and running diskutil list — you should see an EFI partition with at least a few hundred MB used on the USB disk.

Re-run OCLP → Build and Install OpenCore, select the USB drive again, and try a different USB port on the Mac Pro. All four USB 3.0 ports on the back are bootable.

OpenCore picker appears but "Install macOS Sonoma" is not listed

The installer USB isn't being detected. Try: unplug and re-plug the installer USB into a different USB port, then wait 10 seconds. If it still doesn't appear, the installer USB was not created correctly — recreate it using OCLP's Create macOS Installer tool.

Confirm the installer USB has an entry in diskutil list showing a large partition labeled "Install macOS Sonoma" before booting.

System is very slow after installation — sluggish UI, blurry fonts

This is software-rendering mode — your FirePro GPUs do not yet have their drivers. This is normal and expected before root patching. Boot fully into Sonoma, open OCLP, click Post-Install Root Patches, and run the patching. Reboot. The UI will be normal afterward.

If patching was already run and the system is still slow, check: did the reboot after patching happen via the OpenCore picker? Booting without OpenCore may cause the patches to not take effect.

Black screen after reboot following root patching

Power off completely (hold the power button for 5 seconds, or unplug). Wait 15 seconds. Power back on. A hard power cycle lets the Mac Pro's SMC fully reset, which sometimes resolves GPU initialization stalls.

Try a different display port. For external displays, try HDMI first — it has a simpler signal chain than Thunderbolt. If using a TB2→DisplayPort or TB2→HDMI adapter, try a different adapter or a direct Thunderbolt connection.

If still black: hold Option at startup and select OpenCore, then in the OpenCore picker press Space on the Sonoma entry and choose verbose mode (-v). The boot log will show where the GPU driver is failing.

OCLP root patching fails — "Cannot mount system volume" or similar error

This usually indicates a snapshot issue. Try from Terminal:

Terminal
% diskutil apfs listSnapshots /
# If multiple snapshots are listed, note the most recent

% sudo bless --mount / --bootefi --create-snapshot
# Then re-try OCLP root patching

Also check that you have at least 3 GB of free space on the system volume. OCLP needs room to stage the kexts and rebuild the cache.

iMessage or FaceTime says "Contact Apple Support to Activate"

This is common after major macOS reinstalls. Steps to fix:

1. Sign out of Apple ID completely: System Settings → Apple ID → Sign Out. Restart. Sign back in.

2. If still failing: in OCLP → Utilities → Find the iMessage repair option. This regenerates correct ROM and MLB identifiers matching your MacPro6,1 board's original values. Run it, then sign out and back into Apple ID.

3. Wait up to 24 hours after a fresh Apple ID sign-in — Apple's activation servers sometimes require time to verify re-used hardware.

After a macOS Sonoma update, the system is slow again (no GPU acceleration)

This is expected and correct behavior. Every Sonoma update re-seals the system volume, removing your root patches. Open OCLP → Post-Install Root Patches → Start Root Patching → Reboot. GPU acceleration returns immediately.

If root patching fails after an update, it's likely you need a newer OCLP version that supports the new Sonoma build. Check github.com/dortania/OpenCore-Legacy-Patcher/releases for a release that lists your new Sonoma build version.

Kernel panic / system crash after applying patches

A kernel panic on a freshly patched system usually means an OCLP version mismatch. Check your exact macOS build: run sw_vers in Terminal and compare the BuildVersion against OCLP release notes. Download the correct OCLP version, re-patch, and reboot.

If panics persist, boot from your backup clone and re-assess. Running a clean Sonoma install immediately followed by root patching (before any other changes) is the most reliable way to establish a stable baseline.

Wi-Fi or Bluetooth stops working after Sonoma install

The original BCM94360 Wi-Fi/Bluetooth card in the MacPro6,1 is natively supported by Sonoma and requires no additional patches. If Wi-Fi is missing:

Toggle it off and on: System Settings → Wi-Fi → toggle the switch. If still missing, check OCLP's Post-Install patches — sometimes a Wi-Fi patch is included for BCM cards; run it and reboot.

If you have replaced the original Wi-Fi card with a third-party card, additional kexts may be needed. The standard guide covers only original hardware.

Thunderbolt 2 devices not recognized

Try a cold boot with the Thunderbolt device already connected. The Mac Pro sometimes needs the device present at boot to initialize the TB2 controller correctly under OCLP.

Confirm the device works with a known-good TB2 cable. If using a TB2→TB3 adapter (Apple's $49 adapter), connect first and then hot-plug the TB3 device — don't connect them together before plugging in.

How do I revert to macOS Ventura if things go wrong?

Boot from your backup clone or Time Machine backup. Ventura is natively supported on the Mac Pro 6,1 — no OCLP needed. You can also create a Ventura installer USB and do a clean install from it: download the Ventura installer from the App Store, use createinstallmedia to create the USB, and install without any OpenCore involvement.

Removing OpenCore from the internal EFI: mount the EFI partition (sudo diskutil mount disk0s1), open the EFI folder in Finder, delete the OC and BOOT folders under /Volumes/EFI/EFI/, and unmount. The Mac Pro will boot normally from firmware again.
