Setting Up Python on Mac

macOS ships with Python 2.7 (deprecated) and sometimes Python 3 via Xcode tools — but you want a clean, managed install. Here's the right way.

1

Install Homebrew (Mac package manager)

Homebrew is the standard Mac package manager. Open Terminal.app (⌘ Space → "Terminal") and run:

Terminal — zsh
% /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
# Follow the prompts. Enter your password when asked.

After install, follow the instructions to add Homebrew to your PATH (shown in the output).

2

Install Python 3 via Homebrew

Never use /usr/bin/python3 (the system Python) for development. Install your own:

Terminal — zsh
% brew install python
% python3 --version
Python 3.12.3
% which python3
/opt/homebrew/bin/python3
On Mac, always type python3, not python. The bare python command may invoke the old system Python 2.
3

Create a virtual environment

Isolate project dependencies so packages don't conflict across projects.

Terminal — zsh
% mkdir my_project && cd my_project
% python3 -m venv .venv
% source .venv/bin/activate
(.venv) % # prompt shows (.venv) when active
(.venv) % pip install requests
(.venv) % deactivate
4

Write & run your first script

Create a file, write Python, run it. VS Code with the Python extension is the recommended editor on Mac.

Terminal — zsh
% echo 'print("Hello, Mac!")' > hello.py
% python3 hello.py
Hello, Mac!
% python3 # opens interactive REPL
>>> 2 + 2
4
>>> exit()
Mac shortcut: In VS Code, press ⌘⇧P → "Python: Select Interpreter" → choose your .venv. Then ⌃` opens a terminal with the venv already active.

Syntax, Variables & Types

Python uses indentation instead of curly braces. No semicolons. No variable declarations — just assign. Dynamically typed but strongly typed: "1" + 1 raises TypeError.

The Golden Rules

  • indent4 spaces per block level. Never mix tabs and spaces. Python raises IndentationError if inconsistent.
  • no ;Semicolons are optional and discouraged. One statement per line.
  • no {} Blocks are opened by a colon : and closed by dedenting — no braces.
  • #Comments start with #. No block-comment syntax — use multiple # lines.

Type System

intArbitrary precision integer. 42, -7, 1_000_000 (underscores for readability) float64-bit double. 3.14, 1e-4. Division always returns float. boolTrue / False (capital T/F). Subclass of int: True == 1 strImmutable Unicode text. Single, double, or triple quotes. NoneThe null type. Use is None / is not None to test it. complexComplex numbers: 3 + 4j

F-Strings (Python 3.6+)

Prefix a string with f to embed expressions directly. The preferred formatting method.

f"Hello, {name}!" — any Python expression inside {}

f"{price:.2f}" — format spec after :

f"{value!r}" — !r uses repr(), !s uses str()

Operators vs Other Languages

  • //Floor division (integer result). 7 // 2 = 3
  • **Exponentiation. 2 ** 8 = 256
  • and or notLogical operators — NOT && || !
  • isIdentity check (same object). Use == for value equality.
  • inMembership: "a" in "cat" → True
  • :=Walrus operator (Python 3.8+): assign and test in one expression.

Strings

Strings are immutable Unicode sequences. Slice them like lists. Dozens of built-in methods. F-strings are the modern formatting approach.

Three quoting styles

'single' and "double" are identical. Use whichever avoids escaping apostrophes.

"""triple""" — spans multiple lines. Used for docstrings.

r"raw\nstring" — backslashes are literal. Essential for file paths and regex.

Slicing: s[start:stop:step]

Start is inclusive, stop is exclusive. Negative indices count from the end.

s[0] First character s[-1] Last character s[2:5] Characters at index 2, 3, 4 s[:3] First three characters s[-3:] Last three characters s[::-1]Reversed string

Most-used string methods

  • .strip()Remove leading/trailing whitespace (or given chars)
  • .split()Split on whitespace or a given separator → list
  • sep.join()Join an iterable into a string — ", ".join(lst)
  • .replace()Replace all occurrences of a substring
  • .startswith()Returns bool — also .endswith()
  • .find()Returns index of first match or -1
  • .upper() .lower() .title()Case conversion
  • .isdigit()True if all chars are digits — also .isalpha(), .isspace()
Mac path tip: always use raw strings or pathlib.Path for file paths to avoid backslash issues — even on Mac, raw strings prevent \t being read as a tab character.

Numbers & Math

Python ints have arbitrary precision (no overflow). Floats are 64-bit IEEE 754. The math module covers trig, log, and more.

Integer features

Python ints never overflow — they grow as large as memory allows. You can write 1_000_000 with underscores for readability. Integer literals support hex (0xFF), octal (0o77), and binary (0b1010).

Float gotcha

Floating point representation is imprecise: 0.1 + 0.2 ≠ 0.3. Use round() for display, or the decimal module for financial code.

Built-in numeric functions

  • abs()Absolute value
  • round(n, d)Round to d decimal places
  • min() max()Works on any number of args or an iterable
  • sum()Sum an iterable
  • divmod(a, b)Returns (quotient, remainder) as a tuple
  • pow(x, y, mod)Like x**y but with optional modulus
Mac terminal tip: In the Python REPL, use ⌃D to exit, or type exit(). Use ↑↓ to navigate history.

Lists

Lists are ordered, mutable, heterogeneous sequences. The workhorse data structure. Slicing works identically to strings.

Creating lists

[1, 2, 3] — literal. list(range(10)) — from iterable. [0] * 5 — five zeros.

Key methods

  • .append(x)Add x to end — O(1)
  • .extend(it)Add all items from iterable — O(k)
  • .insert(i, x)Insert x before index i — O(n)
  • .pop()Remove & return last item. .pop(i) at index i.
  • .remove(x)Remove first occurrence of x. Raises ValueError if missing.
  • .index(x)Return index of first x. Raises ValueError if missing.
  • .sort()Sort in-place. sorted(lst) returns a new list.
  • .reverse()Reverse in-place. lst[::-1] returns reversed copy.
  • .copy()Shallow copy (same as lst[:])
  • .count(x)Count occurrences of x
  • .clear()Remove all items
Shallow vs deep copy: lst.copy() copies the list but not nested objects. For nested structures use import copy; copy.deepcopy(lst).
lists.py 
# ── Creating lists ────────────────────────────────────
fruits = ["apple", "banana", "cherry"]
numbers = list(range(5))          # [0, 1, 2, 3, 4]
zeros   = [0] * 5                 # [0, 0, 0, 0, 0]
mixed   = [1, "hello", True, None]  # any types

# ── Accessing elements ────────────────────────────────
fruits[0]         # 'apple'
fruits[-1]        # 'cherry'  (last)
fruits[1:3]       # ['banana', 'cherry']
fruits[::-1]      # ['cherry', 'banana', 'apple']
fruits[:2]        # ['apple', 'banana']

# ── Modifying lists ───────────────────────────────────
fruits.append("date")            # add to end
fruits.insert(1, "avocado")      # insert before index 1
fruits.extend(["elderberry", "fig"])  # add multiple
fruits[0] = "APPLE"              # replace by index

# ── Removing items ────────────────────────────────────
fruits.pop()           # removes & returns last item
fruits.pop(0)          # removes & returns index 0
fruits.remove("banana")  # removes first occurrence
del fruits[1]          # delete by index (no return value)

# ── Querying ──────────────────────────────────────────
len(fruits)            # number of items
"apple" in fruits      # True/False
fruits.count("apple")  # occurrences
fruits.index("cherry") # index (raises if missing)

# ── Sorting ───────────────────────────────────────────
nums = [3, 1, 4, 1, 5, 9, 2]
nums.sort()                       # in-place (modifies nums)
nums.sort(reverse=True)           # descending in-place
sorted(nums)                      # returns NEW sorted list
sorted(nums, reverse=True)        # descending new list

# Sort by custom key — sort words by length:
words = ["banana", "apple", "cherry", "fig"]
words.sort(key=len)               # ['fig', 'apple', 'banana', 'cherry']
words.sort(key=lambda w: w[-1])   # sort by last letter

# Sort list of dicts by a field:
people = [{"name": "Bob", "age": 30}, {"name": "Alice", "age": 25}]
people.sort(key=lambda p: p["age"])  # youngest first

# ── Unpacking ─────────────────────────────────────────
a, b, c = [1, 2, 3]
first, *rest = [10, 20, 30, 40]  # first=10, rest=[20,30,40]
*init, last  = [10, 20, 30, 40]  # init=[10,20,30], last=40

# ── Nested lists (2D) ─────────────────────────────────
matrix = [[1, 2, 3],
          [4, 5, 6],
          [7, 8, 9]]
matrix[1][2]   # 6  (row 1, col 2)

# Flatten a 2D list with comprehension:
flat = [x for row in matrix for x in row]  # [1,2,3,4,5,6,7,8,9]

Tuples & Sets

Tuples are immutable lists — use for data that shouldn't change, return values, and dict keys. Sets are unordered collections of unique values with O(1) membership testing.

Tuples

Created with () or just commas: point = 3, 4. Immutable: you can't change elements after creation.

Single-element tuple needs a trailing comma: (42,) not (42).

Named tuples from collections.namedtuple give field names to tuple positions — lightweight struct.

Sets

{1, 2, 3} — set literal. set() — empty set (not {} which is a dict!).

  • |Union — all elements from both sets
  • &Intersection — only elements in both
  • -Difference — in left but not right
  • ^Symmetric difference — in one but not both
  • <= >=Subset / superset check
When to use a set: deduplication (list(set(dupes))), fast membership testing over large collections, set algebra on groups of items.
tuples_sets.py 
from collections import namedtuple

# ── Tuples ────────────────────────────────────────────
point = (3, 4)               # tuple literal
point = 3, 4                 # parentheses optional
single = (42,)               # single-element tuple — NEEDS comma
empty  = ()                  # empty tuple

# Unpacking
x, y = point                 # x=3, y=4
lat, lon, *_ = (37.77, -122.41, 0, 0)  # ignore extras

# Tuples as dict keys (lists can't be)
locations = {(37.77, -122.41): "San Francisco"}

# Named tuple — lightweight struct
Point = namedtuple("Point", ["x", "y"])
p = Point(3, 4)
p.x, p.y      # 3, 4
p[0], p[1]    # 3, 4  (still indexable)

# Functions returning multiple values (it's a tuple)
def min_max(numbers):
    return min(numbers), max(numbers)  # returns (min, max)

low, high = min_max([5, 1, 8, 2])   # unpacks the tuple

# ── Sets ──────────────────────────────────────────────
colors = {"red", "green", "blue"}
empty_set = set()            # NOT {} — that's a dict

colors.add("yellow")         # add one element
colors.discard("red")        # remove (no error if missing)
colors.remove("green")       # remove (raises KeyError if missing)
"blue" in colors             # True  — O(1) lookup

# Set operations
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a | b        # {1, 2, 3, 4, 5, 6}  union
a & b        # {3, 4}              intersection
a - b        # {1, 2}              difference (in a, not b)
b - a        # {5, 6}              difference (in b, not a)
a ^ b        # {1, 2, 5, 6}       symmetric difference

a.issubset({1, 2, 3, 4, 5})  # True
a <= {1, 2, 3, 4, 5}         # True (same thing)
a.issuperset({1, 2})          # True

# Deduplication — fastest way to get unique values
words  = ["apple", "banana", "apple", "cherry", "banana"]
unique = list(set(words))    # order not guaranteed

# frozenset — immutable set (can be a dict key)
fs = frozenset({1, 2, 3})
d = {fs: "immutable set as key"}   # works!

Dictionaries

Dicts map keys to values with O(1) average lookup. In Python 3.7+ they maintain insertion order. The most versatile data structure.

Access patterns

d[key] — raises KeyError if missing.

d.get(key) — returns None if missing.

d.get(key, default) — returns default if missing.

d.setdefault(key, default) — insert if missing, always return value.

Iterating

  • d.keys()View of all keys
  • d.values()View of all values
  • d.items()View of (key, value) pairs — use in for loops

Useful stdlib extras

collections.defaultdict — auto-creates missing keys.

collections.Counter — counts hashable objects.

collections.OrderedDict — preserves order (now built-in since 3.7).

Empty dict vs empty set: {} creates an empty dict, not a set. Use set() for an empty set.
dicts.py 
from collections import defaultdict, Counter

# ── Creating dicts ────────────────────────────────────
person = {"name": "Alice", "age": 30, "city": "San Francisco"}
empty  = {}
also   = dict(name="Alice", age=30)     # keyword constructor
from_pairs = dict([("a", 1), ("b", 2)]) # from list of tuples

# ── Accessing ─────────────────────────────────────────
person["name"]               # 'Alice'  — KeyError if missing
person.get("email")          # None     — safe
person.get("email", "N/A")   # 'N/A'   — with default
"age" in person              # True  (key membership)
"Alice" in person.values()   # True  (value search, O(n))

# ── Modifying ─────────────────────────────────────────
person["email"] = "alice@mac.com"   # add or update
del person["city"]                  # remove key
removed = person.pop("age")         # remove + return value
person.pop("missing", None)         # safe pop with default
person.update({"age": 31, "job": "Engineer"})  # merge in

# ── Iterating ─────────────────────────────────────────
for key in person:                  # iterate keys
    print(key)

for key, val in person.items():     # iterate pairs — preferred
    print(f"{key}: {val}")

keys   = list(person.keys())        # ['name', 'email', 'age', ...]
values = list(person.values())

# ── Merging dicts (Python 3.9+) ───────────────────────
defaults = {"theme": "dark", "lang": "en"}
settings = {"lang": "fr", "font": "Fira Code"}

merged = defaults | settings        # {'theme':'dark','lang':'fr','font':'Fira Code'}
defaults |= settings                # in-place merge

# ── Dict comprehension ────────────────────────────────
squares = {n: n**2 for n in range(1, 6)}   # {1:1, 2:4, 3:9, 4:16, 5:25}
inverted = {v: k for k, v in squares.items()}

# Filter: only even squares
even_sq  = {n: n**2 for n in range(10) if n % 2 == 0}

# ── defaultdict — auto-creates missing keys ───────────
groups = defaultdict(list)
for name, dept in [("Alice","Eng"),("Bob","Eng"),("Carol","HR")]:
    groups[dept].append(name)   # no KeyError on first append
# groups = {'Eng': ['Alice', 'Bob'], 'HR': ['Carol']}

word_count = defaultdict(int)
for word in "the cat sat on the mat".split():
    word_count[word] += 1       # no KeyError on first +=

# ── Counter — count things ────────────────────────────
c = Counter("abracadabra")          # counts characters
c.most_common(3)    # [('a', 5), ('b', 2), ('r', 2)]
c["a"]              # 5
c["z"]              # 0  (Counter returns 0, not KeyError)

votes = Counter(["Alice", "Bob", "Alice", "Alice", "Bob"])
votes.most_common(1)  # [('Alice', 3)]

if/elif/else ternary match/case truthiness

If / Elif / Else & Pattern Matching

Standard conditional flow. elif (not else if). Python 3.10 adds match/case — structural pattern matching.

Truthiness rules

Anything is truthy except: None, False, 0, 0.0, "", [], {}, set(), ().

So if my_list: tests "is the list non-empty?" — idiomatic Python.

Ternary expression

value_if_true if condition else value_if_false

Reads naturally: "x if condition else y". Good for simple cases; avoid nesting.

match / case (Python 3.10+)

Structural pattern matching — more powerful than a switch statement. Patterns can destructure data, match types, and include guard clauses.

▶ Short-circuit evaluation

a and b — if a is falsy, b is never evaluated. Returns a.

a or b — if a is truthy, b is never evaluated. Returns a.

Common idiom: name = user_input or "default" — uses "default" if input is empty.

control_flow.py 
# ── if / elif / else ─────────────────────────────────
score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "F"

# ── Truthiness ────────────────────────────────────────
items = []
if items:                   # False — empty list is falsy
    print("has items")

name = ""
if not name:                # True — empty string is falsy
    name = "Anonymous"

value = None
if value is None:           # CORRECT way to check None
    print("no value")

# ── Ternary (conditional expression) ─────────────────
label = "even" if score % 2 == 0 else "odd"
status = "pass" if score >= 60 else "fail"
clamped = min(max(score, 0), 100)   # clamp between 0-100

# ── Chaining comparisons (Pythonic!) ─────────────────
x = 5
0 < x < 10          # True  — equivalent to 0 < x and x < 10
0 <= x <= 100        # True
# Don't write: x > 0 and x < 10  (works but verbose)

# ── Short-circuit — 'or' as default ──────────────────
username = ""
display_name = username or "Anonymous"   # 'Anonymous'

config = {"debug": True}
port = config.get("port") or 8080        # 8080

# ── match / case (Python 3.10+) ──────────────────────
command = "go north"

match command.split():
    case ["quit"]:
        print("Quitting...")
    case ["go", direction]:            # capture 'direction'
        print(f"Going {direction}")
    case ["get", item] | ["pick", "up", item]:  # OR pattern
        print(f"Getting {item}")
    case ["attack", enemy] if enemy != "boss":   # guard
        print(f"Attacking {enemy}")
    case _:                            # wildcard — default
        print("Unknown command")

# match on type and structure
point = (1, 2)

Loops

Python's for iterates directly over any iterable — no index arithmetic needed. while loops on a condition. Both support else clauses (unusual but powerful).

Iteration tools

  • range()range(n) → 0 to n-1. range(a,b) → a to b-1. range(a,b,step)
  • enumerate()Yields (index, value) pairs. Use instead of range(len(lst))
  • zip()Iterates multiple iterables in parallel, stops at shortest
  • reversed()Iterate in reverse without copying
  • sorted()Iterate in sorted order without modifying original
  • iter() / next()Manual iteration over any iterable

for…else

The else clause on a for or while runs when the loop completes normally (without break). Useful for "search and not found" patterns.

Pythonic iteration: if you find yourself writing for i in range(len(lst)): lst[i] — that's C-style Python. Use for item in lst or for i, item in enumerate(lst) instead.

Comprehensions & Generators

Comprehensions are a Pythonic one-liner for filtering/transforming iterables. Generator expressions are the lazy, memory-efficient version.

Anatomy

[expression for item in iterable if condition]

Three parts: the output expression, the loop, and an optional filter condition.

Generator expressions

Replace [] with (): (x**2 for x in range(1000)). Produces values lazily — nothing computed until consumed. No memory overhead.

Pass directly to functions that accept iterables: sum(x**2 for x in range(100)) — no list created.

When to use comprehensions vs loops: use comprehensions for straightforward transformations/filters. Stick to regular for-loops when the logic is complex, has side effects, or needs more than one line to express clearly.
▶ Nested comprehensions

Nested loops in a comprehension read left-to-right: outer loop first, inner loop second — same order as writing them out.

[f(x, y) for x in a for y in b] is equivalent to:
result = []
for x in a:
    for y in b:
        result.append(f(x, y))

# ── Generator function with yield ─────────────────────
def squares_gen(n):
    """Lazy generator — produces one value at a time."""
    for i in range(n):
        yield i ** 2       # suspends here, resumes on next()

    print(sq)

# yield from — delegate to another iterable
def chain_gen(*iterables):
    for it in iterables:
        yield from it       # yield each item from it

Functions

Functions are first-class objects in Python — they can be stored in variables, passed as arguments, and returned from other functions. def creates a function. lambda creates an anonymous one.

Argument types

  • positionalRegular arguments matched by position
  • defaultdef f(x, y=10) — y is optional. Defaults evaluated once at definition.
  • *argsVariadic positionals — collected into a tuple
  • keyword-onlyArgs after *: def f(*, key) — must pass by name
  • **kwargsVariadic keywords — collected into a dict
  • pos-onlyArgs before /: def f(x, /) — cannot pass by name (Python 3.8+)

Mutable default gotcha

Never use a mutable object as a default: def f(lst=[]): — the same list is reused across all calls.

Correct pattern: def f(lst=None): lst = lst or []

Closures

A function that captures variables from its enclosing scope. The inner function "closes over" the outer variable — it persists even after the outer function returns.

Docstrings: the first string literal inside a function body becomes func.__doc__. Use triple-quoted strings. Access with help(func) in the REPL.

    # Class variable — shared by ALL instances
    kingdom = "Animalia"

        # Instance variables — unique per object
        self.name  = name
        self.sound = sound
        self.legs  = legs
        self._secret = 42     # _ = "private by convention"

        return f"{self.name} says {self.sound}"

        return f"{self.name} ({self.legs} legs)"

    def __eq__(self, other) -> bool:   # animal1 == animal2
        if not isinstance(other, Animal):
            return NotImplemented
        return self.name == other.name

    def create_dog(cls) -> "Animal":
        return cls("Dog", "Woof", 4)

    @staticmethod                       # receives nothing
    def describe() -> str:
        return "Animals are multicellular organisms"

Decorators

A decorator is a function that receives a function and returns a modified version. @deco above def f() is shorthand for f = deco(f).

Why decorators exist

They let you add cross-cutting behaviour (logging, caching, timing, retrying, auth checks) to functions without modifying their code. Apply one decorator to 10 functions instead of writing the same try/except block 10 times.

Important built-in decorators

  • @staticmethodNo implicit first arg. Not bound to instance or class.
  • @classmethodReceives class as first arg (cls). Used for alternate constructors.
  • @propertyMethod accessed as an attribute. Computed read-only attributes.
  • @functools.lru_cacheMemoize: cache results of expensive calls.
  • @functools.wrapsPreserve wrapped function's name and docstring.
  • @dataclassAuto-generate __init__, __repr__, __eq__.
Always apply @functools.wraps(func) inside your decorator. Without it, every decorated function appears as "wrapper" in tracebacks and help().
decorators.py 
import functools
import time

# ── Simple decorator ──────────────────────────────────
def log_calls(func):
    @functools.wraps(func)       # preserves func.__name__, __doc__
    def wrapper(*args, **kwargs):
        print(f"→ Calling {func.__name__}")
        result = func(*args, **kwargs)
        print(f"← Done")
        return result
    return wrapper

@log_calls
def greet(name):
    return f"Hello, {name}!"

greet("Alice")   # prints "→ Calling greet" then "← Done"

# ── Timing decorator ──────────────────────────────────
def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"{func.__name__} took {elapsed:.4f}s")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(0.1)

# ── Decorator with arguments ──────────────────────────
def retry(max_attempts=3, exceptions=(Exception,)):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == max_attempts - 1:
                        raise
                    print(f"Attempt {attempt+1} failed: {e}. Retrying...")
        return wrapper
    return decorator

@retry(max_attempts=3, exceptions=(ConnectionError,))
def fetch_data():
    pass   # your network code here

# ── Stacking decorators ───────────────────────────────
@timer      # applied second (outer)
@log_calls  # applied first (inner)
def add(a, b):
    return a + b

# Equivalent to: add = timer(log_calls(add))

# ── functools.lru_cache — memoization ─────────────────
@functools.lru_cache(maxsize=128)
def fibonacci(n: int) -> int:
    """Cached recursive fibonacci — fast because of cache."""
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

fibonacci(100)          # instant — would be impossibly slow without cache
fibonacci.cache_info()  # CacheInfo(hits=98, misses=101, ...)
fibonacci.cache_clear() # clear the cache

# Python 3.9+: @functools.cache (unbounded, simpler)
@functools.cache
def factorial(n):
    return 1 if n <= 1 else n * factorial(n - 1)

Exception Handling

Python uses exceptions for error handling. Catch specific exceptions — catching bare Exception hides bugs. The else clause runs only on success; finally always runs.

Full structure

try → except → else → finally

else runs if no exception was raised.
finally always runs — for cleanup (close files, connections).

Best practices

  • Specific firstPut more specific exceptions before general ones.
  • as eUse as e to bind the exception object and access its message.
  • raiseBare raise re-raises the current exception without losing the traceback.
  • raise … from …Chain exceptions: raise NewError("msg") from original_err
Never write except: (bare) or except Exception: without re-raising or logging. Silent exception swallowing hides bugs that are very hard to find later.

# ── Full structure ────────────────────────────────────
def safe_divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("Cannot divide by zero")
        return None
    except TypeError as e:
        print(f"Wrong types: {e}")
        return None
    else:
        # Only runs if no exception was raised
        print(f"Success: {a} / {b} = {result}")
        return result
    finally:
        # Always runs — cleanup code goes here
        print("--- divide attempt done ---")

Always use with open() — guarantees the file is closed. Use pathlib.Path instead of string paths. On Mac, paths start with /Users/yourname/.

Mac-specific paths

File modes

  • "r"Read (default). Raises FileNotFoundError if missing.
  • "w"Write — creates or overwrites.

Modules, Packages & pip

A module is any .py file. A package is a directory with an __init__.py. Import with import or from … import.

Import styles

import math Whole module — access as math.sqrt() import math as m Alias — access as m.sqrt() from math import sqrt, pi Specific names into namespace from math import *Everything — avoid, pollutes namespace

pip on Mac

Always use pip3 or activate your venv first. Never run sudo pip3 — it modifies the system Python.

Mac tip: after Homebrew install, you may need to use pip3 instead of pip. Inside an activated venv, both point to the same thing.

# ── re — regular expressions ──────────────────────────
text = "Call us: (415) 555-1234 or (650) 555-9876"

re.search(r"\d+", text).group()           # '415'
re.findall(r"\(?\d{3}\)?[\s-]\d{3}-\d{4}", text)  # all phone numbers
re.sub(r"\d", "X", text)                  # replace all digits with X
re.split(r"\s+", "  hello   world  ")     # ['', 'hello', 'world', '']

phone_re = re.compile(r"\d{3}-\d{4}")
phone_re.findall(text)

# ── asyncio — basic async/await pattern ──────────────
async def fetch(url: str) -> str:
    """Simulate async network fetch."""
    await asyncio.sleep(0.1)     # non-blocking wait
    return f"data from {url}"

async def main():
    # Sequential:
    data = await fetch("https://api.example.com")

    # Concurrent (run both at once):
    results = await asyncio.gather(
        fetch("https://a.com"),
        fetch("https://b.com"),
        fetch("https://c.com"),
    )
    print(results)

asyncio.run(main())    # entry point for async code
